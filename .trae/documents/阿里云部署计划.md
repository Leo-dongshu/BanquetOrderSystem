# 宴席信息管理系统 - 阿里云服务器部署计划

## 一、项目概述

本项目是一个基于 Vue 3 + Express + MySQL 的全栈应用，需要部署到阿里云服务器上。

## 二、部署步骤

### 1. 准备工作

#### 1.1 服务器选购

- 推荐配置：2核4G，40GB系统盘
- 操作系统：Ubuntu 22.04 LTS
- 安全组设置：开放端口 22 (SSH)、80 (HTTP)、443 (HTTPS)、3306 (MySQL)、8082 (后端)

#### 1.2 域名配置（可选但推荐）

- 在阿里云控制台购买域名
- 配置域名解析，将域名指向服务器公网IP

### 2. 服务器环境配置

#### 2.1 连接服务器

```bash
ssh root@your-server-ip
```

#### 2.2 更新系统

```bash
apt update && apt upgrade -y
```

#### 2.3 安装 Node.js

```bash
# 安装 nvm (Node Version Manager)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc

# 安装 Node.js 18 LTS
nvm install 18
nvm use 18
```

#### 2.4 安装 MySQL

```bash
# 安装 MySQL 8.0
apt install mysql-server -y

# 启动 MySQL 服务
systemctl start mysql
systemctl enable mysql

# 安全配置
mysql_secure_installation

# 创建数据库
mysql -u root -p
```

在 MySQL 中执行：

```sql
CREATE DATABASE banquet_order_system CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'banquet_user'@'localhost' IDENTIFIED BY 'your_strong_password';
GRANT ALL PRIVILEGES ON banquet_order_system.* TO 'banquet_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

#### 2.5 安装 Nginx

```bash
apt install nginx -y
systemctl start nginx
systemctl enable nginx
```

### 3. 后端部署

#### 3.1 上传项目文件

```bash
# 在本地执行
# 使用 git 或 scp 上传项目到服务器
git clone https://github.com/Leo-dongshu/BanquetOrderSystem.git
cd BanquetOrderSystem
```

#### 3.2 配置后端环境

```bash
cd backend
cp .env.example .env
nano .env
```

修改 `.env` 文件：

```env
PORT=8082
NODE_ENV=production

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=banquet_user
DB_PASSWORD=your_strong_password
DB_NAME=banquet_order_system

# JWT 密钥配置
JWT_SECRET=your-super-secret-jwt-key-change-in-production
JWT_EXPIRES_IN=7d
```

#### 3.3 安装依赖并构建

```bash
npm install
npm run build
```

#### 3.4 使用 PM2 管理后端进程

```bash
npm install -g pm2
pm2 start dist/main.js --name banquet-backend
pm2 save
pm2 startup
```

### 4. 前端部署

#### 4.1 修改前端 API 地址

```bash
cd ../frontend
nano src/api/index.ts
```

将 API 地址修改为服务器地址：

```typescript
const api = axios.create({
  baseURL: 'http://your-server-ip:8082/api', // 或使用域名
  timeout: 10000,
});
```

#### 4.2 构建前端项目

```bash
npm install
npm run build
```

#### 4.3 配置 Nginx

```bash
nano /etc/nginx/sites-available/banquet
```

添加以下配置：

```nginx
server {
    listen 80;
    server_name your-domain.com; # 或服务器IP

    # 前端静态文件
    location / {
        root /path/to/BanquetOrderSystem/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://localhost:8082;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

#### 4.4 启用配置

```bash
# 创建软链接
ln -s /etc/nginx/sites-available/banquet /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重载 Nginx
systemctl reload nginx
```

### 5. 配置 HTTPS（推荐）

#### 5.1 使用 Certbot 申请免费 SSL 证书

```bash
apt install certbot python3-certbot-nginx -y
certbot --nginx -d your-domain.com
```

#### 5.2 自动续期

```bash
certbot renew --dry-run
```

### 6. 防火墙配置

```bash
# 安装 ufw
apt install ufw -y

# 允许端口
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp

# 启用防火墙
ufw enable
```

### 7. 备份配置

#### 7.1 数据库备份脚本

创建 `backup-db.sh`：

```bash
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/root/backups"
mkdir -p $BACKUP_DIR
mysqldump -u banquet_user -p'your_strong_password' banquet_order_system > $BACKUP_DIR/banquet_$DATE.sql
# 保留最近7天的备份
find $BACKUP_DIR -name "banquet_*.sql" -mtime +7 -delete
```

#### 7.2 设置定时任务

```bash
crontab -e
```

添加：

```
0 2 * * * /root/backup-db.sh
```

## 三、注意事项

1. **安全性**：
   - 确保密码强度足够
   - 定期更新系统和软件包
   - 配置防火墙限制访问
   - 使用 HTTPS 加密数据传输
2. **性能优化**：
   - 配置 Nginx 缓存
   - 优化数据库查询
   - 启用 gzip 压缩
3. **监控**：
   - 使用 PM2 监控后端服务
   - 监控服务器资源使用
   - 配置日志轮转

## 四、验证部署

1. 访问 `http://your-server-ip` 或 `https://your-domain.com`
2. 测试登录功能
3. 测试订单创建和管理
4. 验证数据库连接正常

## 五、常见问题排查

1. **后端服务无法启动**：检查 PM2 日志 `pm2 logs banquet-backend`
2. **数据库连接失败**：检查 .env 配置和 MySQL 用户权限
3. **前端无法访问后端**：检查 Nginx 配置和防火墙设置
4. **HTTPS 证书问题**：检查 Certbot 配置和域名解析
