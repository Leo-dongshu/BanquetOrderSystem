# 订单状态管理设计

## 1. 数据库设计

### 1.1 订单状态维表 (order_statuses)

* 用于存储所有可能的订单状态
* 字段：
  * `id` (INTEGER, 主键)：状态ID
  * `name` (VARCHAR(50))：状态名称
  * `description` (TEXT)：状态描述
  * `created_at` (DATETIME)：创建时间
  * `updated_at` (DATETIME)：更新时间

* 初始数据：
  | ID | 名称 | 描述 |
  |----|------|------|
  | -1 | 已取消 | 客户退订 |
  | 1 | 待安排 | 新建订单默认状态 |
  | 2 | 已安排 | 人员安排完成 |
  | 3 | 待回款 | 服务完成 |
  | 9 | 已完成 | 尾款回收 |

### 1.2 订单状态历史表 (order_status_history)

* 用于存储订单状态的变更历史
* 字段：
  * `id` (INTEGER, 主键)：记录ID
  * `order_id` (INTEGER, 外键)：关联订单ID
  * `status_id` (INTEGER, 外键)：状态ID
  * `created_by` (VARCHAR(50))：操作人
  * `created_at` (DATETIME)：状态变更时间

### 1.3 订单表 (orders) 修改

* 更新字段：
  * 将 `status` 字段类型改为 `INTEGER`，存储当前状态ID
  * 移除 `staff_arranged` 字段，使用状态ID统一管理
  * 添加外键关联到 `order_statuses` 表的 `id` 字段

## 2. 后端实现

### 2.1 模型定义

* 创建 `OrderStatus` 模型
* 创建 `OrderStatusHistory` 模型
* 更新 `Order` 模型

### 2.2 状态管理逻辑

* 新建订单：
  * 设置初始状态为 1 (待安排)
  * 插入状态历史记录

* 人员安排完成：
  * 更新订单状态为 2 (已安排)
  * 插入状态历史记录

* 服务完成：
  * 更新订单状态为 3 (待回款)
  * 插入状态历史记录

* 尾款回收：
  * 更新订单状态为 9 (已完成)
  * 插入状态历史记录

* 客户退订：
  * 更新订单状态为 -1 (已取消)
  * 插入状态历史记录

### 2.3 API 接口

* 获取订单状态列表
* 获取订单状态历史
* 更新订单状态

## 3. 前端实现

### 3.1 状态显示逻辑

* 维护状态映射表：
  ```javascript
  const statusMap = {
    '-1': { name: '已取消', type: 'danger' },
    '1': { name: '待安排', type: 'warning' },
    '2': { name: '已安排', type: 'success' },
    '3': { name: '待回款', type: 'info' },
    '9': { name: '已完成', type: 'success' }
  };
  ```

* 订单管理页面：显示当前状态和状态变更历史
* 人员安排页面：根据订单状态显示不同的操作选项

### 3.2 状态管理界面

* 订单详情页面添加状态变更历史记录
* 提供状态转换的操作界面
* 显示状态变更的时间和操作人
